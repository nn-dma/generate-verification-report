Conclusion

The following activities were performed during the verification of IT_SOLUTION_NAME release PIPELINE_RUN_ID in the ENVIRONMENT environment:

  1. Approval of IT change for implementation.
  2. Peer review and approval of all changes to requirements.
  3. Peer review and approval of all changes to basis for configuration management including traceability to requirements.
  4. Peer review and approval of all test cases for both installation verification and acceptance verification.
  5. Peer review of all code written as part of the release, including production code, test code, infrastructure code, and pipeline code.
  6. Installation of the IT solution in the ENVIRONMENT environment.
  7. Acknowledgement that any manual configurations required for the release have been executed.
  8. Execution of installation verification test cases, both automatic and manual ones.
  9. Regression test of the IT solution by execution of all automatic acceptance verification test cases used by previous releases.
  10. Execution of acceptance verification test cases, both automatic and manual ones related to the new release.
  11. Generation of this verification report.

All peer reviews and corresponding approval activities (steps 2 to 5) are documented on the pull request that initiated the release pipeline (see here for details).

All acknowledgements of manual configuration (step 6) and verification (steps 8 and 10) are documented on the pipeline run that generated this verification report (see here for details).

It is hereby concluded that IT_SOLUTION_NAME release PIPELINE_RUN_ID is fit for intended use and ready to be released for IS_READY_FOR.

The conclusion is based on:

  • All verification activities were successfully executed, and no issues were found.
  • Regression testing was performed by re-executing all previously defined automated test cases, and all such test cases passed.
  • Based on all test cases passing (both for regression testing and for verification of current release) and the performed peer review of documentation, code, and pipeline definitions, the performed verification is assessed to be sufficient.
  • Requirements that only have test cases using verification method Review by exception are concluded not to have been verified. All other requirements are concluded to have been verified.
